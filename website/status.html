A 3270 Terminal & Telnet Framework in java

 [Home] [Status] [SF Project]

Status:

Emulation of IBM-3278 terminal models 2-E through 5-E is currently in development. Currently the Erase/Write and Write commands have partial support, and most Orders (any that do not apply to field starts or attributes) have been implemented. Essentially we can write characters to a partition, and view the results with a subclass of DefaultTerminalRenderer. This is a big first step. Now we need to properly support 3270 attributes in the object model, and spice up the Renderer. Note: There is still no keyboard or output suport.

The generic Telnet NVT implementation is funtional. It consists of a functioning TelnetClient which supports pluggable abstractions for telnet Options. Along with this, for the NVT device there is a 80x24 character buffer implemented in DefaultTerminalModel, which can be rendered to a JTerminal with a DefaultTerminalRenderer. Basic I/O in the form of alphanumeric (ASCII) keys, CRLF, and Backspace has been implemented as a pluggable KeyMap.

The following RFC's have been implemented in a client-side capacity.

  • Telnet NVT - RFC 854
  • Telnet Binary Mode - RFC 856
  • Echo - RFC 857
  • Suppress GA - RFC 858
  • TerminalType - RFC 884
  • End Of Record - RFC 885
  • TN3270 Regime - RFC 1041
  • TN3270e - RFC 1647 (Basic TN3270e Only)

The following functions / features are missing from the above RFC's

The TN3270e option does not (yet) support the following:

  • Binding to a named device, rather than the generic pool.
  • Association of printer devices.
  • The following function implementations (support is planned)
    • SCS_CTL_CODES
    • DATA_STREAM_CTL
    • RESPONSES
    • BIND_IMAGE
    • SYSREQ
